# Запись на прием
Рамки. Система управления записями.

Уровень. Задача, определенная пользователем.

Основной исполнитель. Администратор системы.

Заинтересованные лица и их требования:
- Администратор системы. Хочет быстро и точно оформить запись, минимизировать ошибки при вводе данных.
- Пациент. Хочет записаться на удобное время, получить подтверждение записи.
- Врач. Хочет видеть свое расписание, избегать двойных записей.
- Информационная система регистрации. Хочет вести учет записей, формировать отчетность, оптимизировать загрузку специалистов.

Предусловия. Администратор системы идентифицирован и аутентифицирован.

Результаты (Постусловия):
- Данные о записи сохранены
- Расписание врача обновлено
- пациенту отправлено уведомление
- Сформирована статистика записи

Основной успешный сценарий:
1. Пациент обращается для записи на прием
2. Администратор системы создает новую запись
3. Администратор системы вводит данные клиента
4. Информационная система регистрации проверяет доступность клиента в базе
5. Администратор системы выбирает специалиста и услугу
6. Информационная система регистрации показывает доступные временные интервалы
7. Администратор системы выбирает удобное время
8. Информационная система регистрации проверяет доступность времени
9. Администратор системы подтверждает запись
10. Информационная система регистрации сохраняет запись и блокирует время
11. Информационная система регистрации отправляет уведомление клиенту
12. Администратор системы выдает подтверждение клиенту

Расширения:
*3а. Пациент новый в системе
    1. Администратор системы вводит полные данные пациента
    2. Информационная система регистрации создает новую запись клиента

*6а. Нет доступных записей на прием
    1. Информационная система регистрации предлагает альтернативные даты
    2. Администратор системы согласовывает с пациентом другое время

*8а. Выбранное время уже занято
    1. Информационная система регистрации уведомляет о конфликте
    2. Администратор системы выбирает другое время
